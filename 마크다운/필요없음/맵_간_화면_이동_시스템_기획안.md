# 맵 간 화면 이동 시스템 기획안 (초간단 MVP: 숫자키 이동)

## 개요

MVP 개발 단계에서 **UI 없이** 숫자키(1~7)로 방(씬) 간 이동하는 **최대한 단순한** 시스템입니다.

---

## 핵심 요구사항

1. **턴 소모 없음**: 방 이동은 턴을 소모하지 않습니다.
2. **UI 최소**: 버튼/패널 없이 단축키로 이동합니다.
3. **최대한 단순**: 스크립트 1개, 숫자키 입력 → 씬 전환만.

---

## 구조 (초간단)

### 1. 스크립트 1개만

#### `DevHotkeyRoomTransition.cs` (신규 생성)

- **역할**: 숫자키(1~7) 입력 시 즉시 씬 전환
- **배치**: 씬에 배치하지 않아도 런타임에 자동 생성되어 `DontDestroyOnLoad`로 유지
- **안전장치**: 입력필드(TMP/InputField)에 포커스가 있으면 단축키 무시
- **동작 조건**: Editor 또는 Development Build에서만 동작

**위치**: `Assets/Scripts/Ryu/Global/Managers/DevHotkeyRoomTransition.cs`

### 2. 키 매핑

- 1 = `PlayersRoom`
- 2 = `Hallway`
- 3 = `LivingRoom`
- 4 = `Kitchen`
- 5 = `SiblingsRoom`
- 6 = `Basement`
- 7 = `Backyard`

---

### 4. 씬 이름 규칙

**씬 파일 이름 = `GameLocation` enum 값**

| GameLocation | 씬 파일 이름 |
|-------------|-------------|
| PlayersRoom | `PlayersRoom.unity` |
| Hallway | `Hallway.unity` |
| LivingRoom | `LivingRoom.unity` |
| Kitchen | `Kitchen.unity` |
| SiblingsRoom | `SiblingsRoom.unity` |
| Basement | `Basement.unity` |
| Backyard | `Backyard.unity` |

---

## 구현 단계 (3단계)

### 1. 스크립트 생성
- `DevHotkeyRoomTransition.cs` 파일 생성
- `Update()`에서 숫자키 입력 감지 → 씬 전환

### 2. 맵 UI 생성
- (없음) UI를 만들지 않습니다.

### 3. 모든 씬에 추가
- (없음) 런타임 자동 생성이므로 씬별 추가 작업이 없습니다.

---

## 전체 흐름

```
[숫자키 1~7 입력]
  ↓
DevHotkeyRoomTransition: GameLocation 매핑
  ↓
GameStateManager.SetCurrentLocation()
  ↓
SceneFadeManager.LoadSceneWithFade()
  ↓
[페이드 아웃 → 씬 로드 → 페이드 인]
  ↓
[완료]
```
